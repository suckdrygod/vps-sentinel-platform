# Deployment

## Prepare

```bash
git clone https://github.com/suckdrygod/vps-sentinel-platform.git
cd vps-sentinel-platform
./deploy/install.sh
```

The default command:

1. checks Docker and Docker Compose;
2. creates `.env` from `config/example.env` when absent;
3. checks whether the external Docker network exists;
4. checks out pinned child repositories into `sources/`;
5. pulls `ghcr.io/suckdrygod/komari:1.2.6-tg.32`;
6. starts nothing unless a start flag is provided.

The installer does not create the external Docker network unless you explicitly opt in:

```bash
./deploy/install.sh --create-network
```

## Start backend manually

```bash
./deploy/install.sh --start-backend
```

Default URL:

```text
http://127.0.0.1:25874
```

Change the binding in `.env`:

```env
KOMARI_HOST=127.0.0.1
KOMARI_PORT=25874
```

## Optional frontend preview

```bash
./deploy/install.sh --with-frontend
```

This is intended for local preview only. Production should keep using the packaged frontend from the pinned Komari release unless you explicitly know why you need a separate frontend preview.

## Optional agent install command

```bash
./deploy/install.sh --print-agent-command
```

The printed command is a template. Replace the endpoint and token with values generated by your panel.

## Rollback

```bash
./deploy/install.sh --rollback
```

Rollback does not delete data:

- `.env` is preserved;
- `data/` is preserved;
- `sources/` is preserved;
- Docker volumes/directories are preserved.

## Stop platform containers

```bash
./deploy/uninstall.sh
```

To intentionally delete local platform data:

```bash
./deploy/uninstall.sh --purge-data
```
