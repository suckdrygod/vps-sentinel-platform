# Architecture

VPS Sentinel Platform is a coordination repository. It is not a replacement for the existing Komari, TCPing, or Komari Web repositories.

## Components

```text
vps-sentinel-platform
├── deploy/             # Compose and installer entry points
├── config/             # Version and runtime configuration
├── docs/               # Documentation center
└── sources/            # Generated by install.sh; pinned checkouts only
```

## Version-pinned sources

The installer checks out each child repository at a fixed ref and commit:

- Komari Panel: `1.2.5-tg.25`
- TCPing Safe Agent: `v1.2.13-safe.13`
- Komari Web: `radix` branch snapshot `ea27b292360d0c0cc61c0ad66765a761e3d604cb`

The platform repository does not patch child repositories.

## Runtime model

`deploy/docker-compose.yml` defines three profiles:

- `backend`: Komari backend container;
- `frontend`: optional local Komari Web preview;
- `agent`: disabled placeholder only.

No profile starts unless the user explicitly starts it. This avoids accidental interference with an existing production deployment.

## Theme extension point

The reserved theme path is:

```text
/theme/emerald-pro/index.css
```

This is a convention for future packaging. The platform does not force-load it and does not modify Komari Web.
